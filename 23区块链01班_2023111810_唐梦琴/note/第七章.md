面向对象OOP
封装、继承、多态
结构体：值类型
type 结构体名 struct{
    属性1 类型1
    属性2 类型2
    ...
}
同一个包里，结构体名字不能重复
同类型的属性可以写在一行
使用：
1.var 变量名 结构体名
变量名.属性1 = 值1
变量名.属性2 = 值2
...

用new实例化类型本质是一个指针

匿名结构体
直接写数据类型，不需要写属性名，缺点：不能有重复的数据类型
变量名 := struct{
    属性1 类型1
    属性2 类型2
    ...
}{//初始化成员属性}

方法
方法有接受者，而函数没有接受者，方法可以重名，因为针对的是不同的接受者，函数不能重名，其他地方同函数
语法格式
func (接收者变量 结构体名) 方法名(参数列表) (返回值列表) {
    // 方法体
}
调用方法：
变量名.方法名()

函数
func 函数名（参数列表）（返回值列表）{
    // 函数体
}
调用函数：
函数名（参数列表）

接口：一种抽象类型，定义了一组方法的签名，任何实现了这些方法的类型都被认为是实现了该接口
type 接口名 interface{
    方法名1(参数列表1) 返回值列表1
    方法名2(参数列表2) 返回值列表2
    ...
}
实现接口：
type 结构体名 struct{
    属性1 类型1
    属性2 类型2
    ...
}
func (接收者变量 结构体名) 方法名1(参数列表1) 返回值列表1{
    // 方法体
}
func (接收者变量 结构体名) 方法名2(参数列表2) 返回值列表2{
    // 方法体
}
...
